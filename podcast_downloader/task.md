# 実装計画書
　　

## 1. プロジェクトのセットアップ

1.  **ディレクトリ作成:**
    -   音声ファイルを保存するための `downloads` ディレクトリを作成する。
2.  **ファイル作成:**
    -   メインの処理を記述する `podcast_downloader.py` を作成する。
    -   使い方などを記載する `README.md` を作成する。
3.  **ライブラリのインストール:**
    -   ターミナルで以下のコマンドを実行し、必要なライブラリをインストールする。
        ```bash
        pip install feedparser requests
        ```

## 2. RSSフィード解析機能の実装

-   **担当:** `podcast_downloader.py`
-   **内容:**
    1.  `argparse` ライブラリを使用し、コマンドライン引数でRSSフィードのURLを受け取る処理を実装する。
    2.  `feedparser` ライブラリを使用し、受け取ったURLのフィードを解析する関数を作成する。
    3.  フィードから全エピソードのタイトルと音声ファイルのURLを抽出し、リスト形式で保持する処理を実装する。
    4.  URLが無効な場合などのエラーハンドリングを実装する。

## 3. ユーザーインターフェースの実装

-   **担当:** `podcast_downloader.py`
-   **内容:**
    1.  解析したエピソードのリストを、番号を付けてコンソールに出力し、ユーザーに選択を促す関数を作成する。
    2.  `input()` 関数を使い、ユーザーからのキーボード入力を受け付ける。
    3.  入力された番号が有効か（リストの範囲内か）をチェックするバリデーション処理を実装する。

## 4. ダウンロード機能の実装

-   **担当:** `podcast_downloader.py`
-   **内容:**
    1.  ユーザーが選択したエピソードの音声ファイルURLを取得する。
    2.  `requests` ライブラリを使用し、URLから音声データをストリーミングでダウンロードする関数を作成する。
    3.  ダウンロードしたデータを `downloads` ディレクトリに保存する。ファイル名はエピソードのタイトルから生成する。
    4.  ダウンロード中・完了などのメッセージをコンソールに表示する。
    5.  ネットワークエラーやファイル書き込みエラーに対応する例外処理を実装する。

## 5. 全体の統合とテスト

-   **担当:** `podcast_downloader.py`
-   **内容:**
    1.  上記で作成した各機能を、メインの実行ブロック (`if __name__ == "__main__":`) で結合し、全体の処理フローを完成させる。
    2.  実際のポッドキャストRSSフィードをいくつか用意し、以下の項目をテストする。
        -   エピソードリストが正しく表示されるか。
        -   選択したエピソードが `downloads` フォルダに正しく保存されるか。
        -   無効なURLや不正なエピソード番号を入力した際に、適切なエラーメッセージが表示されるか。
